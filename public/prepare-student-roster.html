How to prepare your student roster for class placement

Why your roster file matters more than your algorithm.

Cohort Logic team · July 2026 · 5 min read

The most common reason class placement goes sideways isn't the process, it's the data going into it. A roster with inconsistent formatting, missing fields, or stale information produces class lists that don't reflect what you actually know about your students.

Before you import anything into a class placement tool, or build your own spreadsheet, there are a few things worth cleaning up first.

What to include

At minimum, your roster needs:

  • Student legal name (or the name you'll use on class lists)
  • Grade level
  • Any academic scores you want to balance across classes (reading level, math benchmark, etc.)
  • Program flags: IEP, 504, ELL, Gifted (whatever your school tracks that affects placement)
  • Gender or demographic categories you want to distribute evenly

What to leave out

Don't include information you don't intend to use for placement decisions. Addresses, phone numbers, parent contact information, or Social Security numbers for example, none of that belongs in a class placement file. Keep it to what the algorithm will actually act on as it assists with optimizing for equitable class assignments.

This matters beyond privacy. Extra columns create confusion during the import and mapping process, and increase the chance of accidentally sharing sensitive data with the wrong person.

Formatting that prevents headaches

A few things that break imports or cause mapping errors:

Merged cells
unmerge everything before exporting
Multiple header rows
one row of column names, then data
Mixed formats in the same column
if grade is a number, make it a number everywhere (not "3rd" in some rows and "3" in others)
Blank rows between students
remove them
Special characters in names
most tools handle accents fine, but dashes and unusual punctuation in name fields can cause issues

The column that matters most

Whatever academic metric you're using to balance class averages (such as reading level, a composite score, a benchmark number), make sure it's a number, not a label. "Below grade level" can't be averaged. A score of 1.8 can.

If your data only has labels (Approaching / Meeting / Exceeding), consider assigning numeric values before you import, if you would like to see averages.

Keep a backup

Before you make any edits to your roster file, save a copy with a date in the filename. You'll want to refer back to the original, especially if a parent or teacher questions a placement decision later. The software also allows you to save a new file for future use, if you make changes to configurations or data during your session and would like to start from where you left off at a future date.

The short version
  • ✓Clean data beats a clever algorithm. Fix the roster first.
  • ✓Include only fields you'll actually use for placement.
  • ✓Make your balancing metric a number, not a label.
  • ✓Save a dated backup before you edit anything.
