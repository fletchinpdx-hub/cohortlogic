Security & privacy

Honest about what we do and don't do.

Student data is sensitive. We'll tell you exactly how we handle it — what's true now, what isn't yet, and what we're committed to building toward.

Where we are today

Cohort Logic is not yet SOC 2 audited. We are not enrolled in formal DPA programs. We won't claim those things until they're real. What we can say with certainty: student data in Class Builder never leaves your browser. That's the architecture, not just the promise — and that's our strongest privacy protection right now.

What's true today

Current commitments.

🔒

Student data stays in your browser

Class Builder runs entirely client-side. When you import a spreadsheet or Google Sheet, the data lives only in your browser tab. Nothing is uploaded to Cohort Logic's servers. When you close the tab, it's gone. This is not a setting you configure — it's how the app is built.

🙈

We can't see your student data

Because student data never reaches our servers, we genuinely cannot access it. There are no admin backdoors, no database queries, no logs. This is a deliberate architectural choice made to protect student privacy before we've earned the technical certifications that would justify a different approach.

📢

We'll tell you before anything changes

If we ever change the data architecture — adding server-side storage, syncing, or collaboration features — we'll announce it clearly and give you a meaningful opt-in choice. We won't move student data to servers without telling you it's happening and why.

🔑

Your account data is stored securely

Account information (email, name, school name) is stored in Supabase, a Postgres-based backend with row-level security. Passwords are hashed. We don't store payment data because there is none. Session management uses industry-standard JWT tokens.

🚫

No ads. No data sales. Ever.

We don't run ads. We don't sell user data or behavioral profiles. The only revenue model we're building toward is direct subscription fees from schools that find value in the tools. Full stop.

How to use Class Builder safely

You don't have to import names.

Class Builder supports student ID numbers as an alternative to names. If your school prefers to minimize PII in the tool, here's a guide to what to include and what to leave out.

Data field ✕ Avoid ✓ Prefer
Student identity Full legal name + date of birth Student ID number or preferred name only
Grade level Specific homeroom + teacher name Grade level only (K, 1, 2 …)
Academic scores Raw test score + test name + date Normalized 1–5 scale or percentile band
Special education Disability category, service provider, diagnosis Boolean IEP: Yes/No only
Demographics Specific ethnicity labels from your SIS Broad category codes (e.g. A, B, C) for balancing only
Contact info Phone numbers, addresses, parent names Leave out entirely — not needed for balancing

The app only uses what you give it. The leaner your import, the lower your exposure risk — even in a browser-only tool.
